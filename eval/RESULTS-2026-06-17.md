# Eval results — 2026-06-17 (first real test)

First evaluation on varied material instead of a single hand-crafted sentence. 7 fixtures:
4 positives (one **real** public-domain text — Lippmann, *Public Opinion* 1922 — plus three
on-pattern probes, one per lens) and 3 negative controls (recipe, science exposition, sports recap).

## Local backend (`huihui_ai/qwen2.5-abliterate:14b`, Ollama, JSON-mode)

| metric | result |
|---|---|
| positives fired | **1/4** |
| negatives quiet | **2/3** |
| marker coverage | **1/4** |

- Only the explicit permeation probe fired (gradualism + permeation, idx 36.75 "notable", correct markers).
- **Under-fired badly:** the Lippmann "manufacture of consent" passage, the dense adept-register probe, and the AI-persuasion-copy probe all returned **zero** markers. On real/varied text the local 14b is unreliable — the earlier "0→3 hits" was one easy, cue-saturated sentence.
- **One false positive:** the sports recap fired `gradualism` (idx 25.1) — caught at the "incidental" tier by the grader's breadth gate (one marker ≠ a finding), so it did not become a false "notable," but the detection step still emitted noise.

## Cloud backend (OpenRouter, `google/gemini-2.5-flash`)

Routed via **OpenRouter** (the bias study's channel; key in `~/.claude/agents/.env`) — not the
Anthropic SDK.

| metric | result |
|---|---|
| positives fired | **4/4** |
| negatives quiet | **3/3** |
| marker coverage | **4/4** |

Flawless on this set. Lippmann → `manufactured_consent` (idx 25.9, correctly "incidental" — one
marker isn't a finding); the permeation probe → `gradualism`+`permeation` (notable); the adept
probe → all six initiatory markers (idx 78.7, "heavy adept register"); the AI-copy probe →
`personalized_persuasion`+`engagement_steering`+`cognitive_offload` (notable). All three negatives
fired **nothing** — zero false positives, correct tiers throughout.

## Verdict

**Detection works on the default (cloud) path: 4/4 positives, 0 false positives, correct markers and
tiers.** The taxonomy, prompt, and grader are validated end to end. The **local** path is the only
weak component (1/4 here) — its tuning TODO below stands, but it is the degraded, uncensored
*fallback* for filter-tripping material, not the default. `backend="auto"` is therefore correct:
cloud first, local only when the cloud refuses.

## Tuning TODO (driven by this eval)

1. **Run the cloud eval** where a key exists; that is the real quality baseline.
2. **Strengthen the local prompt:** per-marker worked examples (not just a gold-span list), an explicit "fire decisively when a cue clearly matches," and possibly the model's **native tools API** (the 14b advertises `tools`) instead of free-form JSON-mode.
3. **Two-pass local detection:** cheap cue/keyword prescreen → model judges only the candidate markers (less for it to reason over at once → less under-firing).
4. **Expand the eval set** — more real documents per lens, more negatives, and the dossiers' non-gold source paragraphs.
5. **Re-run after each change**; this harness is the regression gate.
