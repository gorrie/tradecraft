# Eval results — 2026-06-28 (full text-lens coverage)

Second real evaluation. The 2026-06-17 set had 7 fixtures over 3 lenses; this expands to **19
fixtures over all 7 text lenses**, with a real/canonical known-positive *and* a cue-saturated probe
per lens, plus 5 negative controls spanning multiple lenses. Two graph lenses (`revolving_door`,
`network_brokerage`) are validated separately by `tests/test_structural.py` and `tests/test_network.py`
against the real entity graph — they read topology, not prose, so they are not eval fixtures.

Each positive is tagged `min_backend`: `cues` (cue-saturated — must fire on the deterministic floor)
or `llm` (real quote or paraphrase — needs the model; SKIPPED on the cues run, asserted on cloud).

## Cloud backend (OpenRouter, `google/gemini-2.5-flash`)

| metric | result |
|---|---|
| positives fired | **14/14** |
| negatives quiet | **5/5** |
| marker coverage | **14/14** |

Clean sweep. Every text lens fires on both its cue probe and its generalization fixture, and stays
quiet on all five negative controls. Notable generalization wins (no literal cue present):

- `legibility_scott_paraphrase` → `legibility_imperative`, `standardization_grid` (idx 49.5)
- `counterprod_illich_paraphrase` → iatrogenic_reversal + radical_monopoly + threshold + disabling (83.38)
- `distacct_paraphrase` → nobody_decides, moralized_dissent (57.69)
- `inev_techno_paraphrase` → tina, adapt_or_die (71.67)
- `inst_lippmann_real` (Lippmann 1922, verbatim) → manufactured_consent
- `inev_thatcher_real` ("There is no alternative") → tina

This is a large improvement over 2026-06-17, where cloud was partial and the cue-saturated probes
were the only reliable firings.

## Deterministic `cues` floor (offline, no key, no model)

| metric | result |
|---|---|
| positives fired | **9/9** (the `min_backend: cues` set) |
| negatives quiet | **5/5** |
| marker coverage | **9/9** |
| skipped (need model) | 5 (the paraphrase/canonical fixtures) |

The cues floor catches every cue-saturated probe across all 7 text lenses and stays silent on all
negatives — **zero false positives**, a clean improvement over the 2026-06-17 local 14b, which fired
`gradualism` on a sports recap. The floor cannot (and is not asked to) fire on paraphrased material;
that is the model's job. The floor's role is a cheap, re-runnable CI gate and pre-filter.

## Local backend (Ollama abliterate)

Not re-measured this round. The 2026-06-17 finding stands: the stock abliterated local model
under-fires badly on varied prose (1/4). For reliable prose detection use `cloud` or `auto`
(cloud→local fallback); the `cues` floor is the dependable offline path. Improving local-model
recall (two-pass detection) remains open — see the repo's gap list.

## How to reproduce

```
python eval/run_eval.py cues     # offline floor, no key
python eval/run_eval.py cloud    # full validation (needs OPENROUTER_API_KEY or ~/.claude/agents/.env)
python eval/run_eval.py auto     # cloud, falling back to local
```
